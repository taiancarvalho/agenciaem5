1 O que é o em5

A premissa: você não opera os clientes — você opera a agência. Os agentes operam os clientes.

🎯 Para quem

Gestor solo (ou time enxuto) de agência digital que atende múltiplos clientes e precisa escalar sem inflar headcount.

⚙️ Como roda

Em cima do Claude Code (CLI). Cada agente é um arquivo. Cada cliente é uma pasta. Cada decisão vira documento.

⏱️ Orçamento

Toda interação humana tem teto de 5 minutos. Se passou de 5, é bug de UX — vira ticket de refator.

Regra de ouro Nada vai ao cliente sem veredicto APROVADO do @qa (Crivo). Não tem "publicar assim mesmo".

2 Setup inicial — 5 passos

Da clonagem ao primeiro cliente operando. Estimativa: 30 min na primeira vez.

1

Instalar pré-requisitos

Node 20+, npm, git, jq. Também o Claude Code (CLI da Anthropic).

node --version npm i -g @anthropic-ai/claude-code
2

Clonar o em5

Trazer o repositório pra sua máquina e entrar na pasta.

git clone <repo> em5 && cd em5
3

Configurar Composio (obrigatório)

Único gateway externo. Liga Meta Ads, Google Analytics, Gmail, Slack, Sheets. Pega o token no painel do Composio e exporta como variável de ambiente.

export COMPOSIO_API_KEY=...
4

Rodar /setup

Wizard interativo. Pergunta nome da agência, seus dados, canais que opera, e personaliza os agentes (fantasy names).

/setup
5

Criar primeiro cliente

Gera o workspace do cliente em .em5/clientes/{slug}/ com toda estrutura.

/cliente-novo nome-do-cliente /onboard nome-do-cliente
Pronto. Depois disso, dia-a-dia é só rodar skills (/campanha, /relatorio, /dia) e os agentes fazem o resto.

3 Organograma da agência

15 agentes, cada um com um papel. Você fala com @ceo (Atlas). Ele traduz pro @coo (Nexus). Nexus delega.

Liderança (você → CEO → COO)
Operação (executam o cliente)
Meta-time (constrói o em5)
Gate final (não passa sem)

4 Fluxo padrão — anatomia de uma campanha

Do "vamos lançar uma campanha" ao anúncio rodando, em 7 estações.

Por que tantas estações? Cada agente toca apenas o domínio dele (Constituição Art. II). @copywriter não escolhe verba. @traffic não reescreve oferta. Isso é o que mantém qualidade alta sem você no meio.

Filesystem first Você NUNCA acha info em chat. Sempre em arquivo. Se um agente "lembrou" algo sem ter arquivo, é bug — escala para o @ceo.

7 Princípios inegociáveis

Os artigos da Constituição que o sistema enforça automaticamente.

📁 Art. I — Filesystem First

Toda decisão vira documento. Conversa é exploração; arquivo é verdade. Estado vive em /clientes/{slug}/.

🎯 Art. II — Autoridade do Agente

Cada agente = 1 domínio. Não invade alheio. @copywriter não decide verba; @traffic não reescreve oferta.

📦 Art. III — Artefato sobre Conversa

Handoff entre agentes sempre referencia um documento. Nunca "baseado no que conversamos".

🔍 Art. VIII — Quality Gate

Nada sai sem veredicto do @qa. Lint mecânico (/lint-pre-qa) pré-filtra, mas Crivo é a barreira final.

🔌 Art. IX — Composio Gateway

Todo acesso a serviço externo passa por Composio MCP. Sem credencial solta, sem integração ad-hoc.

⏱️ Art. XII — em Cinco Minutos

Toda interação humana ≤ 5 min. Passou de 5? Bug de UX. Vira ticket de refator via /construir.

8 FAQ iniciante

Dúvidas que toda primeira semana tem.

Por onde começo no dia-a-dia?

Roda /dia (quando estiver na Fase 1.4) ou /status nome-cliente pra cada cliente ativo. Em ≤ 5 min você sabe o que precisa atenção. Daí decide qual skill rodar.

Preciso conversar com cada agente individualmente?

Não. Você fala com @ceo (Atlas). Ele entende intenção, traduz pra @coo (Nexus), e Nexus delega pros executores. Skills (/campanha, /relatorio) já fazem essa orquestração sem você acionar agente por agente.

Como o em5 não inventa dado?

Composio é o único gateway externo. Agentes só puxam métricas via Meta Ads API / Google Analytics (reais). Se algo não tem dado, o agente registra "sem dado disponível" no markdown — ele não fabrica número.

O que acontece se eu pular o /lint-pre-qa?

O @qa (Crivo) vai cobrar lint. Se não houver arquivo lint-YYYY-MM-DD.md do dia, ele devolve pedindo lint primeiro. Crivo não valida char count — Crivo valida gancho, oferta, coerência. Lint é pré-filtro mecânico.

Posso "publicar assim mesmo" mesmo com BLOQUEADO?

Não, e não é aleatório: o gate é constitucional (Art. VIII). Se você tem motivo forte, usa /override (planejado pra Fase 3.3) que registra a decisão + auditoria. Mas o default é: BLOQUEADO = bloqueado.

E se eu quiser um agente novo (ex: SEO)?

Não escreve direto. Roda /construir agente-seo. O meta-time (@arq → @builder → @reviewer) entrevista, projeta, implementa e valida contra a Constituição. Garante coerência com o resto do sistema.

Quanto custa rodar o em5?

Custo principal: tokens do Claude (LLM). Agentes usam tier frontier (Opus) para decisões críticas (@ceo, @qa, @reviewer, @arq) e balanced (Sonnet) para execução. Composio tem plano gratuito + pago conforme volume de chamadas.

Onde tá a documentação completa?

Três arquivos:

• docs/INSTALL.md — passo-a-passo de instalação técnica
• docs/MANUAL.md — manual completo de uso
• .em5/core/constitution.md — os 13 artigos inegociáveis

9 Próximos passos

Você tá instalado. Cliente criado. E agora?

1. Onboard o cliente

Rode /onboard {slug}. O @cs vai te guiar pra coletar briefing em ≤ 5 min.

2. Estratégia

O @strategist (Vera) lê o briefing e gera plano-estrategico.md automaticamente. Você valida.

3. Primeira campanha

Rode /campanha {slug}. Em 5 min você tem copy, criativo, lint, QA e estrutura pronta pra subir.

4. Daily run

Próxima rotina: /dia de manhã. Sistema mostra desvios, sugere otimizações.

5. Relatório mensal

Fim do mês: /relatorio {slug} gera HTML pro cliente e envia via Gmail.

6. Aprende com o sistema

Toda decisão importante vira aprendizado em .em5/learnings/. O próximo cliente já parte mais inteligente.

Lembrete Você está operando a agência, não os clientes. Confia nos agentes, eles foram desenhados pra fazer o trabalho. Seu papel é dar intenção e validar pontos de virada.
