# Agency OS — Flow 5: CRO (Conversão)

# Objetivo

Definir como a agência otimiza a conversão após o clique, garantindo que tráfego gerado se transforme em leads qualificados, vendas e receita.

---

# Posição no sistema

```
Tráfego → gera clique
CRO → melhora conversão
CS/Vendas → transforma conversão em receita
```

---

# Papel do CRO

## Função principal

Melhorar continuamente a taxa de conversão das páginas, mensagens e experiências do usuário.

## O que ele faz

- analisa páginas, landing pages, checkout e fluxos de WhatsApp
- identifica pontos de fricção
- propõe testes
- melhora headline, estrutura e CTA
- analisa comportamento do usuário
- trabalha junto com copy e design
- documenta aprendizados dentro dos logs já existentes do sistema

---

# Anti-papel

O CRO não deve:

- rodar tráfego
- escrever copy isoladamente
- criar design sem base
- atuar sem dados
- criar um sistema paralelo de documentação

---

# Regra principal

```
Só otimiza com base em dados e hipótese clara
```

---

# Workflow macro

```
1. Coletar dados
2. Diagnosticar problemas
3. Criar hipóteses
4. Definir testes
5. Executar variações
6. Medir resultados
7. Documentar aprendizados
8. Iterar
```

---

# Etapa 1 — Coleta de dados

## Fontes

- taxa de conversão
- CPL ou CPA
- taxa de clique
- comportamento do usuário
- leitura do gestor de tráfego
- leitura do CS, quando houver interação relevante

---

# Etapa 2 — Diagnóstico

## Perguntas-chave

- onde o usuário abandona?
- o problema é oferta, copy, layout ou experiência?
- há fricção?
- existe incoerência entre anúncio, página e CTA?

---

# Etapa 3 — Hipóteses

## Exemplo

```
Trocar a headline pode aumentar conversão
Adicionar prova social pode reduzir objeção
Simplificar CTA pode aumentar resposta
```

---

# Etapa 4 — Testes

## Tipos

- teste de headline
- variação de CTA
- layout diferente
- ordem de seções
- ajuste de mensagem
- ajuste de prova social

---

# Etapa 5 — Execução

- criar variações com apoio de copy e design
- implementar as mudanças
- deixar o tráfego continuar enviando volume

---

# Etapa 6 — Medição

- comparar resultados
- validar hipótese
- decidir se mantém, descarta ou itera

---

# Documentação do CRO

## Regra

O CRO não cria um log separado.

Os aprendizados e ações de CRO devem ser registrados dentro dos 2 logs oficiais do sistema:

```
1. log-operacional.md
2. log-performance-criativa.md
```

## Como registrar

- mudanças operacionais, follow-ups e pendências → [log-operacional.md](http://log-operacional.md)
- testes de headline, criativo, CTA, página, mensagens e seus resultados → [log-performance-criativa.md](http://log-performance-criativa.md)

---

# Integração

## Copy

- ajusta mensagem

## Design

- ajusta layout

## Tráfego

- envia volume e mede impacto

## CS

- pode registrar retorno qualitativo vindo do cliente ou do lead, quando isso ajudar na leitura

# Resultado

Se bem feito:

- aumenta conversão
- reduz custo por resultado
- melhora ROI
- transforma tráfego em receita com mais eficiência

---

# Limite

CRO não substitui tráfego, copy ou design.

Ele existe para otimizar o que já está rodando, usando os logs já definidos no sistema e sem criar complexidade desnecessária.
