# Agency OS — Flow Copywriter

# Objetivo

Definir de forma completa como o Copywriter opera dentro do Agency OS, transformando briefing, estratégia, ICP e oferta em mensagens que geram ação e conversão.

---

# Posição no sistema

O Copywriter entra depois que:

- o cliente já foi onboardado
- a estratégia já foi definida
- o gestor de tráfego já identificou o que precisa ser produzido

## Papel no ciclo

```
Estratégia → define direção
Gestor de Tráfego → define necessidade operacional
Copywriter → transforma direção em mensagem
Designer → transforma mensagem em visual
Gestor de Tráfego → publica e mede
```

---

# Observação estrutural importante

Tudo que está descrito aqui pertence a um único agente:

## Copywriter

Não faz sentido quebrar esse papel em vários subagentes pequenos.

O Copywriter é um único agente com múltiplas tarefas especializadas.

## Regra estrutural

```
1 agente = 1 domínio de responsabilidade
AGENTE = PAPEL
TASK = EXECUÇÃO
```

---

# Papel do Copywriter

## Função principal

Traduzir estratégia, ICP, oferta e contexto do cliente em mensagens claras, persuasivas e acionáveis para os canais corretos.

## O que ele faz

- lê onboarding, briefing e estratégia
- entende ICP, dores, desejos, objeções e linguagem
- entende oferta, promessa e diferencial
- define ângulos de comunicação
- cria conceitos de campanha
- escreve copy para campanhas
- adapta a mensagem por canal
- entrega material pronto para design e tráfego
- registra peças no log de performance criativa
- aprende com os resultados medidos pelo tráfego

## Anti-papel

O Copywriter não deve:

- definir canal e verba
- substituir a estratégia macro
- subir campanha na plataforma
- escrever sem direção
- depender só de criatividade solta sem estrutura

---

# Fundamentos de repertório

## Referências estruturais

O Copywriter deve operar com forte base de:

- Great Leads
- Eugene Schwartz
- Gary Halbert
- gatilhos mentais
- leitura crítica de ofertas e promessas

## Regra prática

Esse repertório não entra como romantização do papel. Ele entra como base para criar:

- ângulos
- hooks
- estruturas de mensagem
- diferenciação de oferta
- adaptação de linguagem

---

# Workflow macro do Copywriter

```
1. Ler o cliente
2. Entender ICP
3. Entender oferta
4. Definir ângulos
5. Criar conceitos de campanha
6. Escrever copies
7. Adaptar por canal
8. Entregar para design
9. Registrar no log de performance criativa
10. Ajustar com base no feedback de tráfego
```

---

# Etapa 1 — Leitura do cliente

## Objetivo

Entender o que está sendo vendido, para quem e em qual contexto.

## O que o Copywriter precisa ler

- onboarding
- briefing final
- estratégia
- plano de campanha
- posicionamento e narrativa do cliente
- auditoria de tráfego, quando útil

## Regra

O Copywriter não precisa saber tudo do cliente. Ele precisa entender com profundidade o que impacta conversão.

---

# Etapa 2 — Compreensão do ICP

## Objetivo

Entender as pessoas que a mensagem precisa mover.

## O que precisa ser entendido

- dores
- desejos
- objeções
- anseios
- linguagem
- nível de consciência
- motivadores de decisão

## Regra prática

Se o ICP não estiver claro, a copy tende a ficar genérica.

---

# Etapa 3 — Entendimento da oferta

## Objetivo

Saber exatamente o que está sendo vendido e por que alguém compraria isso agora.

## O que o Copywriter precisa responder

- qual transformação essa oferta entrega
- qual benefício principal
- qual promessa central
- qual diferencial percebido
- por que isso importa agora

---

# Etapa 4 — Definição de ângulos

## Objetivo

Definir diferentes formas de abordar a mesma oferta.

## Exemplos de ângulo

- dor
- desejo
- oportunidade
- prova social
- urgência
- mecanismo único
- conveniência
- autoridade

## Saída esperada

O Copywriter deve gerar 3 a 5 ângulos quando fizer sentido.

---

# Etapa 5 — Criação de conceitos de campanha

## Objetivo

Definir a ideia central antes da copy final.

## O que é um conceito

Headline, promessa e direção principal da peça.

## Exemplo

"Aprenda inglês pagando R$ 13 por dia"

## Regra

O conceito precisa ser forte o suficiente para guiar a produção visual e a publicação.

---

# Etapa 6 — Escrita da copy

## Objetivo

Escrever as mensagens finais que serão usadas na operação.

## O que pode ser escrito

- anúncios
- roteiros de vídeo
- textos de criativo
- legendas
- landing pages
- mensagens de WhatsApp
- e-mails
- textos de LinkedIn
- textos de blog
- SEO de produto, quando fizer sentido

## Estrutura base

```
Hook
→ Desenvolvimento
→ Prova
→ CTA
```

---

# Etapa 7 — Adaptação por canal

## Objetivo

Transformar a mesma ideia em formatos corretos para cada canal.

## Regra

Mesma ideia não significa mesmo formato.

## Templates por canal

```
templates/copy/
  instagram.md
  facebook.md
  whatsapp.md
  email.md
  linkedin.md
  blog.md
  landing-page.md
```

## O que cada template deve conter

- estrutura recomendada
- tamanho ideal
- objetivo do canal
- exemplo de uso

---

# Etapa 8 — Handoff para design

## Objetivo

Entregar material claro para o Designer transformar em peça visual.

## Handoff

```
Copywriter → Designer
```

## O que deve ser entregue

- texto
- ideia visual sugerida
- intenção da peça
- ângulo utilizado
- código único da peça, quando aplicável

---

# Etapa 9 — Log de performance criativa

## Objetivo

Criar uma memória viva do que foi produzido e do que performou.

## Estrutura do arquivo

```
/clientes/cliente-x/operacao/log-performance-criativa.md
```

## Regra

Esse log não substitui o log operacional do cliente.

Ele existe para registrar performance criativa e aprendizado entre copy, design e tráfego.

## Modelo sugerido

```markdown
| ID | Data | Canal | Tipo | Ângulo | Copy | Criativo | Status | Resultado | Observação |
|----|------|------|------|--------|------|----------|--------|-----------|------------|
| CR-001 | 2026-04-05 | Meta | Anúncio | Prova social | "+300 pacientes..." | Video 01 | Ativo | CPL R$12 | Alta conversão |
| CR-002 | 2026-04-06 | Meta | Anúncio | Urgência | "Últimas vagas..." | Image 02 | Pausado | CPL R$45 | Baixa performance |
```

## Campos recomendados

- ID
- Data
- Canal
- Tipo
- Ângulo
- Copy
- Criativo
- Status
- Resultado
- Observação

## Regra de atualização

- Copywriter cria a linha quando a peça nasce
- Gestor de Tráfego atualiza resultado e observação de performance

---

# Etapa 10 — Iteração com base em performance

## Objetivo

Melhorar a escrita com base no que realmente performou.

## O que o Copywriter precisa observar

- quais ângulos performaram melhor
- quais hooks geraram resposta
- quais mensagens morreram
- quais canais responderam melhor a cada abordagem

## Regra

O Copywriter não deve operar no escuro. Ele precisa usar o log de performance criativa para aprender e evoluir.

---

# Nomenclatura e ID único

## Regra

Toda peça relevante precisa ter um código único para conectar copy, design, tráfego e performance.

## Padrão sugerido

```
CR-001 → criativo / peça de copy
CP-001 → campanha
CJ-001 → conjunto
```

## Aplicação prática

Exemplo de anúncio:

```
[CR-001] Prova social - vídeo depoimento
```

## Objetivo

Permitir que a equipe saiba exatamente qual peça está performando e qual copy precisa ser revisitada ou iterada.

# O que este fluxo resolve

Se esse fluxo for bem feito:

- a copy deixa de ser improviso
- a mensagem fica alinhada com estratégia
- o design recebe direção melhor
- o tráfego publica com mais clareza
- a equipe aprende com o que performa
- o sistema passa a acumular inteligência de comunicação

---

# Limite deste fluxo

Este fluxo não deve:

- substituir a estratégia
- decidir canal ou verba sozinho
- ignorar feedback de performance
- operar apenas com talento sem estrutura

Ele existe para transformar comunicação em sistema, repertório em execução e performance em aprendizado contínuo dentro do Agency OS.
